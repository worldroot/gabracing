Exercise 1: Install the Azure Racing Game Lab Prerequisites

The Global Azure Racing Hands-On Lab is based on the XNA Racer game, which is a sample game built using the Microsoft XNA platform. XNA has been deprecated but is available for Visual Studio 2015 and 2017 through the community. In this exercise XNA will be installed on the development PC, and the game solution opened and compiled in Visual Studio to verify that it builds correctly.

 

Task 1: Install the Microsoft XNA Tools and SDK

As the racing game is built using the XNA framework the XNA tools will be required for it to run.

Note: Visual Studio 2015 is required for the installation of the XNA tools. If you do not nave it installed, the free community edition is available here:

https://download.microsoft.com/download/e/4/c/e4c393a9-8fff-441b-ad3a-3f4040317a1f/vs_community.exe

 

Step 1: Install the XNA Tools

 

1.    Download the Visual Studio XNA tools from https://gabdownload.blob.core.windows.net/lab/XNAGameStudio405.zip

 

2.    Extract the ZIP file to a folder.

3.    Open Readme.txt.

4.    Follow the installation instructions, installing the 4 pre-requisites in the numbered folders.

5.    Install XNA Game Studio 4.0.

 

Task 2: Build and Test the Game Solution

The next task is to build and test the game solution. At present the solution is not integrated with Azure and will run as a stand-alone application.

 

 

Step 1: Download and Extract the Racing Game Code

The starting solution for the lab is stores as a ZIP file in Azure blob storage.

1.    Download the Racing Game Start Project from the labs page of the Global Azure Bootcamp Racing Lab site for your region.

2.    Extract the zip file to a location on your local drive.

 

Step 2: Open the Solution and Examine the Code

Open the AzureRacingGameWorkshop.sln solution file and examine the projects.

·         AzureRacingGameWorkshop.AzureData
This contains the placeholder code for integrating with the services hosted in Azure.

·         RacingGame
This contains the racing game implementation.

·         RacingGameContent
This contains the 3D models, textures, sound files and tracks for the game.

·         RacingGameContentPipelineExtension
This contains the content compilation pipeline.

Exercise 2: Test the Game

In order to get a feel of the game, we can test it to see that it works, and gets a suitable frame rate.

 

 

Task 1: Configure your Player Name

To save entering your driver name every time you start the game, the value can be set in the Settings class. This class also contains values for the storage accounts and service bus namespaces that will be used to integrate the driving game with Azure.

1.    In the AzureRacingGameWorkshop.AzureData project, open Settings.cs.

2.    Change the value of the PlayerName variable to your preferred driver name, ensuring that it is unique for all workshop attendees.
